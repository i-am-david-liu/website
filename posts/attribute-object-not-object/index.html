Attribute + object → not object

Non-alcoholic beer.

Beer, by definition, is alcoholic. When you attribute beer as "non-alcoholic", it loses that definition.

But does non-alcoholic beer fall in the category of beer? Certainly, if a drink has "beer" in its name, then it must be some variant of beer…right?

I don't think so.

As I said before, things that fall under the beer category are defined to be alcoholic. So non-alcoholic beer cannot be in the category of beer, or else we'd have a contradiction (a drink cannot simultaneously be alcoholic and non-alcoholic).

Non-alcoholic beer is not beer, despite the name. It is a drink that logically deserves its own class.

Let's look at a typical attribute + object pairing: orange tulip.

Tulips do not have to be orange to be considered tulips. Therefore, orange tulips fall neatly into the category of tulips. Orange tulips are a subset of tulips.

But non-alcoholic beer is not a subset of beer. Isn't that weird?

Here's another example: old news.

News, by definition, describes "recent information". If it's recent, how can it be old? We can try and massage "old news" to mean "information that used to be recent", but then it wouldn't be news. Contradiction!

Q: So? What's the point? These are just oxymorons.

A: Yes, but they're a stricter class of oxymorons. "Jumbo shrimp" is an oxymoron, but it only works as an oxymoron if we refer to "shrimp" as a "thing that's small", rather than "swimming crustacean". Jumbo shrimp, referring to the animal, conceptually holds, because it literally describes "a type of shrimp that's larger than other types of shrimp".

Jumbo shrimp, referring to the nonexistent "big thing that's small", is an oxymoron, and a rather forced one at that. We need to warp the meaning of shrimp in order for the contradiction to emerge, and that lacks elegance and poise.

Meanwhile, non-alcoholic beer is an oxymoron in the most explicit sense. We don't need to swap definitions; the definition is already there, and it is actively being contradicted.
